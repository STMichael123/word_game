# 江湖织梦（StoryWeaver 武侠文字游戏）

一个“NLU 意图识别 + 状态/一致性管理 + LLM 叙事生成”的古风武侠文字冒险 Demo。  
LLM 通过 OpenAI 兼容接口调用（推荐在 `.env` 中配置 `LLM_BASE_URL`、`LLM_MODEL`、`LLM_API_KEY`；旧的 `NVIDIA_*` / `OPENAI_*` 变量也兼容）。

默认推荐使用更大的通用文本模型，例如 `gpt-5.4`。如果你当前的服务商支持更高质量的对话模型，也建议把 `.env` 里的 `LLM_MODEL` 显式设成该模型，而不是依赖较弱或不兼容的默认名。

## 功能概览（扩展版）

- **体量扩展**：主线 8 章节 + 可触发支线任务，目标游玩时长 2-3 小时
- **随机性**：地点事件表、江湖奇遇、战斗结果、掉落、声望涨落（带 seed 可复现）
- **NLU（意图识别）**：规则 + 轻量实体抽取（人物/地点/物品关键词）
- **状态与一致性**：`game_state`（地点、属性、门派/关系、背包、事件史），并做基础矛盾修复（生命值边界、死亡状态、重复物品等）
- **NLG（上下文生成）**：把状态与本轮结果交给 LLM，生成“剧情 + 2~4 个下一步选项”（结构化 JSON）
- **文字战斗场景**：回合制文本战斗（敌人等级随主线进度成长）
- **前端**：水墨风 Web 前端（FastAPI + 原生 HTML/JS，主对话区 + 常驻侧边栏）

## 快速开始（Windows / PowerShell）

1) 创建虚拟环境并安装依赖

```bash
python -m venv .venv
.\.venv\Scripts\Activate.ps1
pip install -r requirements.txt
```

2) 配置环境变量

- 复制 `.env.example` 为 `.env`
- 填入 `LLM_API_KEY`
- 如果你使用第三方 OpenAI 兼容服务，同时确认 `LLM_BASE_URL` 与 `LLM_MODEL` 和你的服务商一致
- 如果没有单独指定模型，项目默认会使用 `gpt-5.4`

3) 运行（水墨风 Web 前端）

```bash
python app_web.py
```

启动后页面顶部会主动探测在线叙事链路。
如果看到“在线叙事未连通”，说明当前密钥、接口地址或模型名至少有一个不匹配，游戏会退回离线兜底叙事。

## 玩法提示（推荐节奏）

- 你可以输入自然语言：`去客栈打听消息` / `探索后山` / `与长老谈判`
- 也可以输入数字选择：`1` / `2` / `3`
- 输入 `/save` 保存，`/load` 读取，`/reset` 重开
- 建议先在一个地点探索 3~5 回合再转场，主线推进更稳定

## 结构

- `storyweaver/`：核心逻辑（NLU/状态/一致性/随机世界表/LLM 客户端）
- `app_web.py`：网页入口（水墨风 Web 版，唯一入口）
